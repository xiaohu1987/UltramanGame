## PLAN 计划拆解

### 目标
重做答题反馈特效（Canvas 粒子 + 音效）：
- **答对**：弹出框慢慢汇聚成光球 → 飞向并击中「要攻击的目标」
- **答错**：弹出框变黑 → 裂开 → 消失

---

### T1: 定位现有答题反馈与攻击目标入口
- **动作**：梳理答对/答错触发链路、反馈弹出框组件、攻击目标节点/坐标来源、现有动画与音效资源位置。
- **交付物**：改动点清单（文件路径 + 关键函数/组件 + 目标坐标获取方式 + 可用音效资源）。
- **依赖**：无
- **验收标准**：能明确指出「答对/答错回调」「弹出框 DOM/渲染层」「攻击目标落点」三处入口，且不依赖猜测。
- **工作量**：小

### T2: 设计答对/答错特效时序与接口
- **动作**：定义统一特效接口与时间轴（汇聚/飞行/命中；变黑/裂开/消失），约定输入参数（弹出框矩形、目标点、对错状态、完成回调），并规划音效触发时机。
- **交付物**：特效规格说明（阶段、时长建议、回调时机、与答题流程衔接方式）。
- **依赖**：T1
- **验收标准**：规格覆盖完整链路；明确「动画期间是否锁定继续答题」；音效与阶段一一对应。
- **工作量**：小

### T3: 实现答对 Canvas 粒子特效（汇聚→光球→击中）
- **动作**：基于 Canvas/粒子实现答对动画：弹出框区域粒子缓慢汇聚成光球，沿路径飞向攻击目标并播放命中反馈；动画结束触发完成回调。
- **交付物**：答对特效模块（可复用函数/组件）+ 命中落点表现。
- **依赖**：T2
- **验收标准**：答对时可见「汇聚→光球→飞向目标→击中」；落点对准攻击目标；结束后可继续后续流程。
- **工作量**：中

### T4: 实现答错 Canvas 粒子特效（变黑→裂开→消失）
- **动作**：实现答错动画：弹出框先变黑/暗化，再裂成碎片粒子并消散；动画结束触发完成回调。
- **交付物**：答错特效模块（可复用函数/组件）。
- **依赖**：T2
- **验收标准**：答错时可见「变黑→裂开→消失」；不残留碎片层；结束后可继续后续流程。
- **工作量**：中

### T5: 接入音效并与动画阶段同步
- **动作**：为答对（汇聚/发射/命中）与答错（变黑/裂开）绑定音效；优先复用项目现有资源，缺失时补充轻量音效或预留可替换路径。
- **交付物**：音效触发点接入 + 资源引用。
- **依赖**：T3、T4
- **验收标准**：答对/答错分别有对应音效；触发时机与关键动画阶段基本同步；无异常重复播放。
- **工作量**：小

### T6: 替换旧反馈并完成联调验收
- **动作**：将新旧特效接入真实答题流程，移除/屏蔽冲突的旧弹出反馈；验证连续答对、连续答错、对错交替、目标位置变化等场景。
- **交付物**：集成后的答题反馈链路 + 验收结果说明。
- **依赖**：T5
- **验收标准**：
  1. 答对：弹出框汇聚成光球并击中攻击目标；
  2. 答错：弹出框变黑裂开消失；
  3. 音效同步且不阻断继续答题；
  4. 无旧特效冲突、无明显卡顿/残留层。
- **工作量**：中

### 关键路径
**T1 → T2 → T3 → T5 → T6**（答对主链路）  
**T4** 与 **T3** 在 T2 完成后可并行，再汇入 T5。

### 可并行任务
- **T3** 与 **T4**（答对/答错特效实现）可并行。
- **T5** 需等 T3、T4 主体完成后再接音效与联调。

### 风险与预案
| 风险 | 影响 | 预案 |
|------|------|------|
| 攻击目标坐标难取（动态移动/多层缩放） | 光球飞偏 | T1 优先确认目标锚点；必要时用 `getBoundingClientRect`/场景坐标转换统一到 Canvas 空间 |
| 旧 CSS/DOM 动画与 Canvas 层冲突 | 双重反馈、层级错乱 | T6 明确替换旧反馈；Canvas 层独立挂载并在结束时销毁 |
| 粒子性能不足（低端机卡顿） | 动画掉帧 | 控制粒子数量与时长；提供降级（减少粒子/缩短阶段） |
| 音效资源缺失或浏览器自动播放限制 | 无声/首次无声 | 复用现有音效；在用户答题交互后触发播放；资源缺失时先保证视觉完整 |
| 动画未结束就进入下一题 | 状态错乱 | T2 约定完成回调；答题流程等待特效结束或设置安全超时 |

### 交付物总览
1. 答对特效：汇聚光球 → 击中攻击目标（Canvas 粒子）
2. 答错特效：变黑 → 裂开消失（Canvas 粒子）
3. 同步音效接入
4. 与现有答题流程联调通过

### 预估总工作量
约 **中等**（核心在 T3/T4/T6，合计约 2–4 小时量级，视现有代码结构而定）
