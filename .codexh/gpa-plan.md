## PLAN：选人页难度弹窗 + 紧凑条（不挡角色）

### 目标回顾
把选人页难度从「常驻大条挡住角色」改为：**每次进入选人页先弹窗选难度**；确认后关闭，页面只保留**不挡角色的紧凑难度条**，规则与持久化不变。

---

### T1: 盘点选人页难度 UI 与遮挡根因
- **动作：** 只读检查 `index.html`、`css/*`（含难度相关样式）、`js/ui.js`、`js/main.js`、`js/math-challenge.js` 中难度条 DOM、布局层级（z-index/position/overflow）与进入选人页的渲染入口。
- **交付物：** 简短根因说明（遮挡元素、触发时机、可复用 API）。
- **依赖：** 无
- **验收标准：** 明确指出当前挡角色的节点/样式；列出弹窗与紧凑条将改动的文件清单。
- **工作量：** 小

### T2: 设计进页难度弹窗交互与状态
- **动作：** 定义弹窗结构（四档难度 + 说明 + 确认）、默认选中（读 `ultraman.difficulty`）、每次进入选人页强制弹出、确认后关闭并写回难度。
- **交付物：** 交互规格（时机、默认值、确认/关闭行为、与紧凑条同步规则）。
- **依赖：** T1
- **验收标准：** 规格覆盖「每次进入都弹」「确认后关闭」「与紧凑条双向同步」；不改变四档规则语义。
- **工作量：** 小

### T3: 实现难度弹窗 DOM 与样式
- **动作：** 在选人页增加模态弹窗（遮罩 + 卡片）；样式与现有战斗风 UI 一致；确保弹窗居中、可点选四档、不依赖旧大条布局。
- **交付物：** `index.html` 弹窗结构 + 对应 CSS（可新建或扩展现有难度样式文件）。
- **依赖：** T2
- **验收标准：** 弹窗视觉完整；四档可点；遮罩盖住选人区但不永久占用布局流；关闭后无残留遮罩。
- **工作量：** 中

### T4: 改造选人页为紧凑难度条（解除遮挡）
- **动作：** 将原常驻大难度条改为顶部/工具区紧凑条（或等价不挡角色布局）；修正 grid/absolute/overflow 导致角色卡片被盖住的问题。
- **交付物：** 更新后的难度条 HTML/CSS；角色网格可完整点击。
- **依赖：** T1（可与 T3 并行设计，落地在 T3 后或同批）
- **验收标准：** 角色卡片区无红色遮挡级覆盖；可正常点选/取消角色；紧凑条仍显示当前难度与简短说明。
- **工作量：** 中

### T5: 接入弹窗逻辑与难度持久化
- **动作：** 在进入选人页流程中调用「打开难度弹窗」；确认时 `setDifficulty` + `localStorage`；紧凑条切换同步同一状态；开战读取同一难度。
- **交付物：** `js/ui.js` / `js/main.js` / `js/math-challenge.js`（按现有职责）中的弹窗开关与同步逻辑。
- **依赖：** T2、T3、T4
- **验收标准：** 每次进入选人页必弹；确认后难度生效并记忆；刷新后默认选中上次难度；紧凑条改难度立即生效。
- **工作量：** 中

### T6: 回归战斗心算拦截与自动战斗
- **动作：** 验证初级跳过心算；中/高/地狱仍在 `resolveAction` 拦截；答对 +5%、答错/超时 Miss/失败；自动战斗同样弹题。
- **交付物：** 简短回归记录（手动路径 + 自动战斗路径）。
- **依赖：** T5
- **验收标准：** 四档规则与改前一致；仅 UI 入口变化，无结算回归。
- **工作量：** 小

### T7: 视觉与交互验收（桌面布局）
- **动作：** 检查选人页：弹窗打开/关闭、紧凑条、角色选满 3/3、开战按钮；确认无横向溢出、无永久遮挡、滚动正常。
- **交付物：** 验收结论（通过/问题列表）。
- **依赖：** T5、T6
- **验收标准：** 用户截图中红框遮挡问题消失；「开战」与选人流程可用。
- **工作量：** 小

---

### 关键路径
`T1 → T2 → T3 → T5 → T6 → T7`  
`T4` 依赖 T1，建议在 T3 后、T5 前完成（或与 T3 紧耦合同批改 UI）。

### 可并行
- **T3 ∥ T4**（在 T1/T2 完成后）：弹窗结构与紧凑条布局可并行推进，合并时统一 class/状态。
- **T6 与 T7** 部分可重叠，但 T7 最终以 T5 完成为准。

### 风险与预案
| 风险 | 影响 | 预案 |
|------|------|------|
| 旧难度条用 absolute/高 z-index 叠在角色网格上 | 改完仍挡角色 | T1 先定位具体选择器；T4 改为文档流紧凑条或降低层级并限制高度 |
| 「每次进入都弹」与紧凑条重复操作体验差 | 用户烦 | 弹窗默认带出上次难度，一键确认即可；紧凑条仅作二次修改 |
| 多处写难度导致状态不一致 | 开战难度错 | 统一单一 `get/setDifficulty` 源；弹窗与紧凑条都走同一 API |
| 只改 HTML 未改进入选人页钩子 | 不弹窗 | T5 明确挂在选人页 show/enter 入口，而非仅首次加载 |
| 回归时误伤战斗拦截 | 心算失效 | T6 固定测 easy skip + normal 弹题两条路径 |

### 明确不做
- 不改四档题目数值、时限、+5%、Miss/失败规则  
- 不重做整页选人 UI / 不新框架  
- 不做「仅首次弹窗」或「完全去掉页面内难度入口」（与已确认选项不符）

### 交付清单（ACT 阶段预期）

2. 不挡角色的紧凑难度条  
3. 难度持久化与开战/心算逻辑保持  
4. 回归通过记录
